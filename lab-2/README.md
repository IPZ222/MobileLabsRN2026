# Лабораторна робота №2: Побудова вкладеної навігації та оптимізація відображення великих списків у React Native із використанням компонентів FlatList та SectionList.

## Мета

Ознайомлення з принципами навігації у мобільних застосунках React Native.

## Опис реалізованого функціоналу

### 1. Списки та оптимізація

В ході роботи реалізовано стрічку з динамічним підвантаженням даних, яке імітує реальні мережеві запити. 
Користувач може оновити стрічку, потягнувши її вниз (Pull-to-Refresh). 
При досягненні кінця списку автоматично запускається підвантаження наступних постів (Infinite Scroll). 
Для структурування інтерфейсу використано кастомні компоненти заголовка (Header), підвалу (Footer) та розділювачі між елементами.
 Для забезпечення високої продуктивності при роботі з великим списком налаштовано параметри віртуалізації: initialNumToRender, maxToRenderPerBatch та windowSize.

### 2. Навігація

- **Вкладена навігація:** Використано структуру Drawer Navigator, всередині якого знаходиться Native Stack Navigator.
- **Динамічні заголовки:** Заголовок екрана деталей автоматично змінюється залежно від назви обраної новини.
- **Кастомне меню:** Створено власний компонент для Drawer, що містить інформацію про автора та зручні переходи.

### Чим відрізняється FlatList від ScrollView?
FlatList а відміну від ScrollView, не рендерить усі елементи одразу.

### Що таке віртуалізація списків?
Це механізм FlatList, де об'єкти списку, які виходять за межі видимості, вивантажуються з пам'яті (або замінюються), а на їх місце при скролінгу підставляються нові дані.

### Як здійснюється передача параметрів між екранами?
Параметри передаються як другий аргумент функції навігації.

### Що таке вкладена навігація?
Це архітектура, де один навігатор знаходиться всередині іншого.

### У яких випадках застосовується SectionList?
SectionList використовується, коли дані мають логічне групування або ієрархію. Наприклад - список контактів.
