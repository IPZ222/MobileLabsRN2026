# Лабораторна робота №4: Робота з файловою системою в React Native з використанням бібліотеки expo-file-system

## Мета
Опанувати механізми роботи з локальною файловою системою мобільного пристрою, використовуючи можливості бібліотеки expo- file-system. Закріпити навички реалізації базових операцій над файлами й папками, організації файлової навігації та аналізу стану файлової системи.

## Основні можливості

### 1. Навігація та перегляд сховища
Інтерактивний провідник – реалізовано динамічний список вмісту директорій з підтримкою вкладеності.
Механіка переходів:
- Вхід у папку здійснюється одиночним натисканням на елемент зі значком 📁.
- Повернення на рівень вгору реалізовано через кнопку "Назад" у навігаційній панелі, яка автоматично блокується в кореневій директорії.
- Breadcrumbs – відображення поточного відносного шляху для зручного орієнтування в структурі папок.

### 2. Робота з файлами та папками
- **Створення нових об'єктів**:
  - Папка – створюється порожня директорія через `makeDirectoryAsync`.
  - Текстовий файл – при створенні можна вказати початковий вміст (необов'язково). Якщо поле залишити порожнім, файл створиться пустим.
- **Видалення** – підтвердження дії через діалогове вікно.

### 3. 3. Вбудований текстовий редактор

Реалізований як повноекранний інтерфейс (Modal) для роботи з файлами формату .txt.
Функціонал:
- Автоматичне зчитування вмісту при відкритті.
- Поле введення з підтримкою багаторядкового тексту (multiline).
- Система збереження змін через `writeAsStringAsync` з наступним оновленням - стану списку.

### 4. Детальна інформація про файл (FileInfo)

При натисканні на кнопку ℹ️ викликається вікно властивостей, що відображає дані, отримані через getInfoAsync:
- **Назва та розширення об'єкта.**
- **Категорія** – автоматичне визначення: "Папка" або тип файлу на основі розширення.
- **Обсяг** – розмір об'єкта, конвертований із байтів у мегабайти (MB).
- **Часовий штамп** – дата та час останньої модифікації у зручному для читання форматі.

### 5. Моніторинг пам'яті

На головному екрані виводяться:
- Загальний обсяг внутрішньої пам'яті.
- Вільний простір.
- Зайнятий простір (різниця між загальним та вільним).
Значення отримуються за допомогою **нового** та **legacy** API `expo-file-system` для уникнення депрекейшн-попереджень.

### 6. Користувацький досвід

- **Візуальна ідентифікація** – використання іконок (📁/📄) для швидкого розрізнення типів контенту.
- **Стан кнопок** – кнопка повернення змінює колір та деактивується, якщо користувач знаходиться в documentDirectory.
- **Зворотний зв'язок** – використання діалогових вікон для інформування про успішні операції або помилки доступу.
- **Безпека інтерфейсу** – використання SafeAreaView для запобігання перекриттю елементів системними вирізами ("чубчиком") та панеллю навігації.

## Інструкція із запуску

1. Клонуйте репозиторій:
   ```bash
   git clone https://github.com/ipz222_ii/MobileLabsRN2026.git
2. **Перейдіть до папки проєкту**
   ```bash
   cd MobileLabsRN2026/lab4
3. **Встановіть необхідні залежності:**
   ```bash
   npm install
4. **Запустіть Expo CLI:**
   ```bash
   npx expo start
5. **Відкрийте додаток:**
   * Скануйте QR-код через додаток Expo Go на вашому смартфоні.

   * Або натисніть **i** для запуску на iOS Simulator.
