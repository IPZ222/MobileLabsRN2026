# Лабораторна робота №6: Побудова авторизації та збереження персональних даних у React Native з використанням Firebase Authentication та Firestore.

## Мета

Набути практичних навичок інтеграції авторизації та обробки персональних даних користувача в мобільному застосунку.

## Опис реалізованого функціоналу

### 1. Автентифікація та безпека

Реалізовано комплексний ігровий елемент, який реагує на широкий спектр дій:
- **Реєстрація:** Створення нового облікового запису за email та паролем з миттєвою перевіркою коректності даних.
- **Вхід:** Авторизація існуючого користувача з перевіркою статусу emailVerified. Якщо email не підтверджено – вхід блокується з відповідним повідомленням.
- **Відновлення паролю:** Користувач може ініціювати скидання пароля через email за допомогою вбудованого методу Firebase sendPasswordResetEmail.
- **Вихід із системи:** Одноразова дія, яка очищає локальний стан та перенаправляє на екран логіну.

### 2. Глобальний стан

Для відстеження статусу користувача впроваджено React Context API:
- **AuthProvider:** Обгортка, яка слухає зміни стану автентифікації через onAuthStateChanged та надає об’єкт поточного користувача усім дочірнім компонентам.
- **Захист маршрутів:** У кореневому _layout.tsx реалізовано умовну навігацію – неавторизовані користувачі або ті, хто не підтвердив email, не мають доступу до захищених екранів групи (app).
- **Loader:** Під час початкового завантаження стану автентифікації відображається індикатор завантаження, що унеможливлює «моргання» екранів.

### 3. Робота з персональними даними 
- **Структура документа:** name, age, city, email, createdAt.
- **Завантаження профілю:** При переході на екран профілю дані зчитуються з Firestore. Якщо документа не існує, він створюється автоматично з порожніми полями.
- **Оновлення даних:** Користувач може змінити ім’я, вік та місто. Збереження відбувається через updateDoc з миттєвим зворотним зв’язком.

## Висновки

У ході виконання лабораторної роботи було реалізовано повноцінну систему автентифікації користувача з використанням Firebase Authentication, що включає реєстрацію з підтвердженням електронної пошти, вхід існуючого користувача, відновлення пароля та вихід із системи. Для зберігання персональних даних налаштовано колекцію Firestore, де документ кожного користувача ідентифікується його uid, а доступ до нього жорстко контролюється як на клієнтському боці, так і через Security Rules. Реалізовано можливість перегляду та оновлення профілю, а також видалення облікового запису з обов’язковою повторною автентифікацією. За допомогою React Context API організовано глобальний стан авторизації, а маршрутизація на базі expo-router забезпечує захист закритих екранів від неавторизованого доступу. Усі операції супроводжуються зрозумілими сповіщеннями та обробкою помилок, що робить застосунок безпечним та зручним для кінцевого користувача.
