# Лабораторна робота №5: Побудова навігації у React Native із використанням бібліотеки Expo Router.

## Мета: 
### Ознайомлення з концепцією file-based маршрутизації в мобільних застосунках.

## Основний функціонал

### Авторизація та безпека

- **Реєстрація та Вхід:** Повноцінні інтерфейси для введення облікових даних із валідацією збігу паролів.
- **Глобальний стан:** Реалізація AuthContext `Context API`, що забезпечує «єдине джерело істини» для стану авторизації в усьому додатку.
- **Захищені маршрути:** Використання механізму Redirect у макетах груп `(auth)` та `(app)` для автоматичного керування доступом.
### Каталог товарів

- **Головний екран:** Список товарів, реалізований через `FlatList` із оптимізованим рендерингом великих масивів даних.
- **Візуалізація:** Сучасний дизайн карток із широкоформатними зображеннями, що займають всю ширину картки, та акцентованою ціною.
- **Динамічні деталі:** Використання хука `useLocalSearchParams` для отримання ідентифікатора товару та відображення специфічного контенту за маршрутом `/details/[id]`.
### Навігація та UI

- **Expo Router:** Файлова маршрутизація, що розділяє додаток на логічні групи: публічну `(auth)` та приватну `(app)`.
- **Глобальні елементи:** Персистентний хедер та футер, винесені в `RootLayout`, які залишаються видимими при навігації між сторінками.
- **Обробка помилок:** Реалізація сторінки `+not-found.tsx`, яка дозволяє користувачу повернутися до безпечного стану при помилці в URL.
- **Юзабіліті:** Кнопка Logout у каталозі для швидкого скидання стану авторизації та перевірки роботи редиректів.

## 📝 Контрольні запитання та відповіді

**1. Яким чином за допомогою Expo Router реалізується перенаправлення неавторизованого користувача?**
Через компонент `<Redirect>`. У файлі `_layout` робиться перевірка: якщо користувач не залогінений, програма автоматично повертає редирект на сторінку входу.

**2. У чому полягає різниця між використанням компонента `<Link>` та метода `router.push()`?**
`<Link>` — це готовий компонент для натискання в інтерфейсі (як посилання в браузері). `router.push()` — це команда в коді, яка спрацьовує після якоїсь дії, наприклад, після успішного натискання на кнопку "Увійти".

**3. Як працюють динамічні маршрути в Expo Router і як отримати передані параметри?**
Створюються файлами з назвою в квадратних дужках, наприклад `[id].js`. Щоб отримати значення цього id у коді, використовується хук `useLocalSearchParams()`.

**4. Чому стан авторизації доцільно зберігати у глобальному контексті (React Context), а не в локальному стані компонента?**
Щоб не передавати дані про авторизацію через усі компоненти вручну. Контекст дає доступ до стану "залогінений чи ні" будь-якому екрану додатка миттєво.

**5. Для чого використовуються групи маршрутів `(folderName)` і як вони впливають на URL-адресу?**
Потрібні для зручної організації файлів та створення різних макетів (Layouts). Головна фішка — назва папки в дужках не відображається в адресному рядку (URL залишається коротким).
